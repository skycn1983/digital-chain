# 🔐 安全审计准备清单

本文档列出安全审计前需要检查和加固的领域。

---

## 📋 审计范围

- **代码审查**: 核心模块、P2P 协议、加密实现
- **漏洞扫描**: 已知漏洞（依赖、算法）
- **渗透测试**: 网络层、API 层、共识层
- **威胁建模**: 攻击向量、风险评级

---

## 🔑 加密安全

### ✅ 已检查

- [x] secp256k1 曲线使用正确（ elliptic 库）
- [x] 私钥长度 32 字节
- [x] 公钥压缩格式
- [x] 地址派生：RIPEMD160(SHA256(publicKey))

### ⚠️ 待加固

- [ ] **私钥存储**：生产环境禁止返回私钥（`/wallet/create` 接口）
- [ ] **随机数生成**：验证 `crypto.randomBytes` 的熵源
- [ ] **侧信道防护**：时间攻击、功耗分析（客户端场景可忽略）
- [ ] **密钥派生**：考虑实现 BIP32/BIP44 分层确定性钱包

### ❌ 已知问题

- `/wallet/create` 返回私钥 - **生产环境必须移除**
- 无 keystore 文件加密（如 PBKDF2 + AES）

---

## 🔐 交易安全

### ✅ 已检查

- [x] 交易哈希计算正确
- [x] nonce 防重放机制（基于地址的 nonce）
- [x] 余额检查（防止双花）
- [x] 交易签名验证（ECDSA）

### ⚠️ 待加固

- [ ] **签名 malleability**：检查签名是否可被篡改（secp256k1 一般安全）
- [ ] **交易过期**：实现 `maxTimestamp` 或区块高度过期
- [ ] **Gas 定价**：防止交易泛洪攻击（最低 GasPrice 检查）
- [ ] **交易池管理**：实现 LRU 淘汰、交易替换（replace-by-fee）

### ❌ 已知问题

- 交易池无大小限制（可能 OOM）
- 无交易优先级排序（按 gasPrice）
- `/transaction` API 在 demo 模式跳过签名验证

---

## 🌐 P2P 网络安全

### ✅ 已检查

- [x] Handshake 协议版本验证
- [x] 自连接检测（防止 loopback）
- [x] 入站连接数限制（maxInbound）
- [x] 出站连接数限制（maxOutbound）
- [x] 心跳机制（ping/pong）
- [x] 消息类型白名单

### ⚠️ 待加固

- [ ] **身份认证**：基于 nodeId 的长期身份，而非 IP 地址
- [ ] **消息加密**：TLS 或 Noise Protocol Framework
- [ ] **DDoS 防护**：连接速率限制、IP 黑名单
- [ ] **分叉攻击防护**：检查区块累积难度的幂等性
- [ ] **交易洪泛攻击**：验证交易签名前限制广播频率

### ❌ 已知问题

- 无 IP 白名单/黑名单机制
- 消息未加密（明文传输）
- 交易验证前已加入内存池（潜在攻击面）

---

## ⛓️ 共识安全

### ✅ 已检查

- [x] 区块哈希计算正确（包含难度目标）
- [x] PoW 难度验证（前导零检查）
- [x] 链上交易双花检查（nonce + balance）
- [x] 分叉选择算法：累计难度最长链

### ⚠️ 待加固

- [ ] **51% 攻击防护**：检查点（checkpointing）机制
- [ ] **时间戳攻击**：中本聪时间窗口限制（未来区块）
- [ ] **难度调整**：当前每10区块调整一次，需验证边界条件
- [ ] **孤块处理**：当前孤儿池大小 100，需监控利用率

### ❌ 已知问题

- 无检查点，51% 攻击后链可完全重写
- 难度调整逻辑简单，可能被操纵
- 分叉重组未验证所有交易的签名（性能优化空间）

## 📦 依赖安全

运行 `npm audit` 检查依赖漏洞：

```bash
cd digital-chain
npm audit --audit-level=moderate
```

**重点关注**:
- `elliptic` - ECDSA 库，需检查版本是否安全
- `express` - Web 框架，需 5.x 版本无已知严重漏洞
- `ws` - WebSocket，需检查 DoS 漏洞

**建议**: 定期更新依赖，使用 `npm outdated` 和 Dependabot

## 🎯 威胁模型

### 攻击向量

| 攻击类型 | 可能性 | 影响 | 缓解措施 |
|---------|--------|------|---------|
| 51% 攻击 | 中 | 高 | 检查点、社区监控 |
| 交易洪泛 | 高 | 中 | 速率限制、交易池大小限制 |
| 女巫攻击 | 中 | 低 | 身份验证（未来） |
| 中间人 | 低（内网） | 高 | TLS 加密（生产） |
| 重放攻击 | 低 | 中 | nonce 机制已防止 |
| 时间戳攻击 | 低 | 中 | 时间窗口验证 |
| 签名伪造 | 极低 | 极高 | secp256k1 安全 |

### 攻击场景

1. **交易洪泛攻击**
   - 攻击者发送大量无效交易
   - 消耗节点 CPU 和内存
   - 缓解：交易验证前先检查格式、nonce、余额

2. **分叉攻击（51%）**
   - 攻击者拥有超过 51% 算力
   - 构造私有链并超过主链
   - 缓解：检查点、监控算力分布

3. **网络分区**
   - 隔离部分节点
   - 创建孤立链
   - 缓解：分叉选择算法（累计难度）
