# 📊 OpenClaw Token (OCT) 代币经济模型

**版本**: v1.0
**日期**: 2026-03-16
**设计目标**: 可持续的价值捕获，激励网络参与，支持生态增长

---

## 一、代币效用（Token Utility）

OCT 是数字链的原生代币，具有以下核心用途：

### 1. 交易费支付
- 所有链上交易必须支付 OCT 作为 Gas 费
- 优先交易可使用更高的 Gas Price 获得打包优先权
- 费用收入分配给矿工（PoW）或验证者（未来 POS）

### 2. 挖矿奖励
- 出块奖励以 OCT 形式发放给矿工
- 提供经济激励维护网络安全
- 奖励金额随时间递减（减半机制）

### 3. 治理代币
- OCT 持有者对协议升级提案进行投票
- 金库资金使用决策
- 参数调整（难度、费用、奖励等）

### 4. 质押机制（未来）
- 质押 OCT 成为验证者（POS 转换后）
- 质押收益来自交易费和通胀奖励
- 惩罚机制（Slashing）防止恶意行为

### 5. 生态支付
- DApp 使用费用（智能合约执行）
- 跨链桥接费用
- 存储费用（IPFS 集成后）

### 释放时间表（Vesting Schedule）

#### 挖矿奖励（50%）
- **初始奖励**: 50 OCT/区块（当前设置）
- **减半周期**: 每 210,000 区块（约 60 天，10 秒/块）
- **释放曲线**:
  ```
  第 1-210,000 块: 50 OCT/块 → 10.5M OCT
  第 210,001-420,000 块: 25 OCT/块 → 5.25M OCT
  第 420,001-630,000 块: 12.5 OCT/块 → 2.625M OCT
  ...
  约 50 年全部释放完毕
  ```
- **预估年通胀率**:
  - 第 1 年: ~10% (基于初始流通量)
  - 第 10 年: ~2%
  - 第 50 年: ~0.1%

#### 生态系统基金（20%）
- **锁定期**: 1 年 cliff（期间不释放）
- **解锁期**: 4 年线性解锁（每年释放 25% = 50M OCT）
- **用途分配**:
  - 50%: 开发者 Grant 和赏金计划
  - 30%: 社区激励（内容创作、翻译、活动）
  - 15%: DApp 生态扶持（空投、流动性挖矿）
  - 5%: 紧急储备

#### 团队（15%）
- **锁定期**: 1 年 cliff（团队继续工作，无代币）
- **解锁期**: 4 年线性解锁（每年 37.5M OCT）
- **归属条件**: 团队持续贡献，离职停止解锁

#### 战略投资者（10%）
- **锁定期**: 6 个月 cliff
- **解锁期**: 2 年线性解锁（每月解锁 1/24）
- **估值**: 私募价格 $0.05-0.10/OCT，融资 $5-10M

#### 流动性储备（5%）
- **立即使用**: 提供初始交易所流动性
- **分配**:
  - 70%: 交易所做市（CEX + DEX）
  - 20%: 社区空投（吸引早期用户）
  - 10%: 紧急流动性

## 三、费用模型（Fee Model）

### 1. Gas 费用结构

**基础费用公式**:
```
总费用 = gasPrice × gasLimit
```

**参数**:
- `gasPrice`: 每 gas 单位价格（OCT），初始 = 1 OCT/gas
- `gasLimit`: 交易最大 gas 消耗
  - 标准转账: 21,000 gas
  - 合约部署: 1,000,000+ gas
  - 合约调用: 50,000-500,000 gas

**示例**: 标准转账费用 = 1 × 21,000 = **21 OCT**

### 2. 动态 Gas 市场（未来升级）

引入 EIP-1559 风格的费用机制:
```
baseFee: 网络自动调整的基础费用（销毁）
priorityFee: 矿工小费（激励优先打包）
maxFee: 用户愿意支付的最高费用

totalFee = max(min(baseFee + priorityFee, maxFee), baseFee)
```

- **baseFee 调整**: 区块满负荷时上升，空闲时下降
- **baseFee 销毁**: 全部销毁，实现通缩
- **priorityFee**: 矿工收入，激励打包

**预期效果**:
- 减少交易费用波动
- 防止垃圾交易（baseFee 门槛）
- 通缩压力（baseFee 销毁）

### 3. 最低费用门槛

**防垃圾交易规则**:
```javascript
MIN_GAS_PRICE = 0.1 OCT/gas
MIN_TOTAL_FEE = 1 OCT
MAX_GAS_LIMIT = 1,000,000 (防止 DoS)
```

## 四、经济参数与机制

### 1. 通胀/通缩模型

**年通胀率计算**:
```
年通胀率 = (年释放量 / 当前流通量) × 100%
```

**当前参数**:
- 初始流通量: 50M OCT（流动性储备 + 生态基金早期解锁）
- 年释放量（挖矿）: 10.5M OCT（第 1 年）
- 年通胀率: 21% （10.5M / 50M）

**长期趋势**:
```
年份  流通量    年释放量  通胀率
0     50M       10.5M     21.0%
1     100M      10.5M     10.5%
2     150M      10.5M     7.0%
3     200M      10.5M     5.25%
...
10    400M      5.25M     1.3%
```

**通缩机制**（EIP-1559 启用后）:
- BaseFee 销毁: 假设 50% 交易费用被销毁
- 年销毁量预估: 2-5M OCT（取决于交易量）
- **净通胀率** = 通胀率 - 销毁率
- 高交易量时可能实现**通缩**（净增量为负）

### 2. 减半时间表

| 减半次数 | 区块高度 | 奖励 (OCT/块) | 累计奖励 | 预计时间 |
|---------|----------|---------------|----------|----------|
| 0 (初始) | 0 | 50 | 0 | 2026-03 |
| 1 | 210,000 | 25 | 10.5M | 2026-05 |
| 2 | 420,000 | 12.5 | 15.75M | 2026-07 |
| 3 | 630,000 | 6.25 | 18.375M | 2026-09 |
| ... | ... | ... | ... | ... |
| 32 | 6,720,000 | ~0.000 | ~999.999M | ~2076 |

**50 年后**: 挖矿奖励接近 0，主要收入来自交易费

### 3. 难度调整

**算法**: 每 10 个区块检查一次
```
目标时间 = 10 × 10 秒 = 100 秒
实际时间 = block[9].timestamp - block[0].timestamp
ratio = 实际时间 / 目标时间

if ratio < 0.9: difficulty++
if ratio > 1.1: difficulty--
```

**作用**:
- 保持出块时间稳定在 10 秒
- 调整矿工竞争难度

## 五、治理机制（DAO）

### 1. 投票权力

**基于持有量的治理**:
- 1 OCT = 1 票
- 委托投票支持（代理投票）
- 快照机制：投票前某个区块高度快照余额

### 2. 提案类型

| 类型 | 描述 | 投票门槛 | 执行延迟 |
|------|------|----------|----------|
| **协议升级** | 修改共识规则、硬分叉 | 总票数 10% + 2/3 通过 | 7 天 |
| **参数调整** | 难度、费用、奖励等参数 | 总票数 5% + 简单多数 | 立即 |
| **金库支出** | 生态系统基金使用 | 总票数 10% + 简单多数 | 3 天 |
| **紧急提案** | 安全漏洞修复 | 基金会 + 2/3 验证人 | 24 小时 |

### 3. DAO 金库

**来源**:
- 生态系统基金（200M OCT）
- 交易费收入分成（5% 转入金库）
- 治理通胀（可选，未来）

**使用**:
- 开发者 Grant（50%）
- 社区活动（20%）
- 流动性激励（20%）
- 运营成本（10%）

### 4. 治理流程

```
1. 提案提交（至少 100k OCT 押金）
2. 讨论期（3 天，Discord/Forum）
3. 投票期（7 天）
4. 结果执行（延迟时间）
5. 如果通过，押金返还；失败，押金销毁 50%
```

## 七、风险评估与缓解

### 1. 经济风险

| 风险 | 描述 | 概率 | 影响 | 缓解措施 |
|------|------|------|------|----------|
| **代币通胀过高** | 挖矿奖励导致供应过剩 | 中 | 高 | 动态调整奖励，加速减半 |
| **流动性不足** | 交易深度差，价格波动大 | 高 | 中 | 预留流动性储备，做市商计划 |
| **价格操纵** | 大户控制价格 | 中 | 中 | DAO 治理限制单笔交易大小 |
| **治理攻击** | 大户垄断投票 | 中 | 高 | 委托投票上限，时间锁 |

### 2. 技术风险

- **智能合约漏洞**: 全面审计，Bug Bounty
- **51% 攻击**: 提高挖矿门槛，考虑 POS 迁移
- **网络拥堵**: 分片或 Layer 2 升级

### 3. 监管风险

- **证券认定**: 避免中心化控制，去中心化治理
- **KYC/AML**: 配合交易所合规要求
- **地域限制**: 限制某些国家访问

**如果出现以下情况，可触发紧急暂停**:
- 重大安全漏洞
- 价格极端波动（>50% 24h）
- 恶意治理攻击

**暂停机制**:
- 多签钱包控制（5/9 签名）
- 暂停后 7 天内社区投票决定后续
- 暂停期间只允许退出，不允许新交易
